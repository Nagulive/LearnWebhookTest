## 2024-06-25 - Contextual Accessibility for Generic Action Buttons
**Learning:** This app frequently relies on list-heavy map-based views (e.g., list of halls) that use generic action button labels like "Book Now" or "Approve." While visually understandable due to proximity to the list item's content, these generic labels provide poor context for screen reader users who may navigate directly between interactive elements or access buttons out of flow. Without explicit `aria-label`s, screen readers announce "Book Now" repeatedly, forcing users to deduce context.
**Action:** Always add contextual `aria-label` attributes to generic buttons in repeating lists or map functions. For example, `aria-label={\`Book ${hall.name}\`}` instead of just relying on the visible "Book Now" text. Additionally, ensure keyboard interactive elements explicitly receive visible focus states (`focus-visible:ring-2`) as this pattern was also missing across these interactive elements.
